
<h3>Суть проблемы и зачем нужен Event Loop</h3>
<p class="info info--blue"><code>JavaScript</code> — однопоточный язык. У него есть только один основной поток выполнения
(Call Stack), а значит, он может делать только одно действие в единицу времени.</p>

<p>
  Если бы тяжелые задачи (сетевые запросы, таймеры) выполнялись синхронно,
  они бы полностью блокировали страницу. Чтобы этого не происходило,
  браузер предоставляет свои инструменты (Web APIs) для фоновой работы, а
  Event Loop (Цикл событий) выступает в роли менеджера, который координирует передачу результатов этих
  фоновых задач обратно в основной поток JS.
</p>

<p>Чтобы понять цикл, нужно знать основные сущности:</p>

<h3>Call Stack</h3>
<p>
  <strong>Call Stack (Стек вызовов)</strong> - это место, где код выполняется прямо сейчас.
  <br>
  Он хранит контексты выполнения (какая функция сейчас выполняется) и примитивные типы данных
  (числа, строки, булевы значения). Стек работает очень быстро и имеет фиксированный размер.
  <br>
  Работает по принципу <b>LIFO (Last In, First Out)</b>.
</p>

<p class="info info--orange">
  Стек вызовов (Call Stack) имеет наивысший и безусловный приоритет.
  <br>
  Поскольку JavaScript — язык однопоточный, движок физически может выполнять только одну операцию в любой момент времени.
  Следовательно, пока есть хоть одна функция в стеке, никакие другие задачи не могут быть выполнены.
  <br>
  Это фундаментальный принцип, который лежит в основе всей модели асинхронности JavaScript.
</p>

<h3>Heap</h3>
<p>
  <strong>Heap (Куча)</strong> — это большая, неструктурированная область оперативной памяти,
  которая используется движком JavaScript для хранения сложных (ссылочных) типов данных.
</p>

<h3>Web APIs</h3>
<p>
  <strong>Web APIs (Фоновые работники)</strong> - это внешние инструменты, предоставляемые средой (браузером).
  <br>
  Когда они заканчивают свою фоновую работу (таймер дотикал, данные с сервера пришли),
  они берут вашу функцию-колбэк и отправляют её в очередь. На этом работа Web API с этой задачей заканчивается.
  Они не хранят очереди внутри себя.
</p>

<h3>Microtasks и Macrotasks (Очереди задач)</h3>
<p>
  Очереди - это просто списки готовых к выполнению функций. Они лежат между Web APIs и движком:
</p>

<p>
  <strong>Microtask Queue:</strong> Очередь с наивысшим приоритетом для отложенного кода.
  <br>
  Сюда попадают колбэки от Промисов (<code>.then</code>, <code>.catch</code>) и <code>async/await</code>.
  <br>
  Важный нюанс: Промисы — это часть самого языка JavaScript, а не Web API,
  поэтому микрозадачи формируются в основном самим движком JS, а не браузером.
</p>

<p>
  <strong>Macrotask Queue:</strong> Очередь с низким приоритетом.
  <br>

  Что сюда попадает: Коллбэки от <code>setTimeout</code>, <code>setInterval</code>, события ввода/вывода,
  обработчики событий UI (клики, скроллы), загрузки скриптов и т.д.
</p>

<h3>Event Loop (Цикл событий) - последовательность выполнения задач</h3>

<p>
  <strong>Очищается Call Stack (Синхронный код):</strong> Выполняется весь обычный JS-код, который сейчас запущен. Стек вызовов должен стать полностью пустым.
</p>

<p>
  <strong>Выполняются ВСЕ Микротаски:</strong> Как только стек опустел, браузер идет в очередь микротасков (промисы). Он выполняет их безостановочно, пока очередь не станет абсолютно пустой (включая те микротаски, которые добавились прямо во время этого шага).
</p>

<p>
  <strong>Рендеринг (Перерисовка UI):</strong> Стек пуст, микротаски выполнены. Браузер оценивает, пришло ли время обновить экран (обычно это происходит ~60 раз в секунду). Если нужно — отрисовывает новые кадры.
</p>

<p>
  <strong>Берется ОДИН Макротаск:</strong> Браузер заглядывает в очередь макротасков (таймеры, клики, сетевые события). Он берет оттуда только одну первую задачу и кидает ее в Call Stack.
</p>

<p>
  <strong>Цикл повторяется:</strong> Возвращаемся к шагу 1 (выполняется код этого макротаска, очищается стек, снова проверяются микротаски и так далее).
</p>

<p class="info info--blue">
  <strong>Краткая формула:</strong>
  <br>
  Весь синхронный код -> ВСЕ микротаски -> Рендер -> ОДИН макротаск -> Повтор
</p>
