<h3>Часткове застосування (Partial Application)</h3>

<p>
  <span class="accent">Часткове застосування</span> — це процес створення нової функції шляхом
  фіксації (попереднього встановлення) одного або кількох аргументів існуючої функції.
</p>
<p>
  Простими словами: у нас є функція, що приймає кілька аргументів. Ми "заморожуємо"
  частину з них і отримуємо нову функцію, яка приймає лише ті аргументи, що залишилися.
</p>

<p class="info info--blue">
  Часткове застосування базується на каруванні, але не вимагає всіх аргументів.
</p>

<code class="code">
  function curry(f) { // curry(f) виконує карування
    return function(a) {
      return function(b) {
        return f(a, b);
      };
    };
  }

  function sum(a, b) {
    return a + b;
  }

  const curriedSum = curry(sum);
  const plusTwo = curriedSum(2); // повертає функцію, яка чекає на другий аргумент

  alert( plusTwo(3) ); // 5
</code>

<h4>Практичне використання часткового застосування:</h4>

<ul>
  <li><strong>Спрощення повторного використання функцій:</strong> Часткове застосування дозволяє створювати спеціалізовані функції на основі більш загальних, що спрощує повторне використання коду.</li>
  <li><strong>Покращення читабельності:</strong> Функції з фіксованими аргументами можуть бути зрозумілішими та зручнішими для читання, особливо якщо вони використовуються в кількох місцях.</li>
  <li><strong>Створення нових функцій із наявних:</strong> Часткове застосування дозволяє легко створювати нові функції, не змінюючи початкову функцію, що сприяє гнучкішому дизайну коду.</li>
</ul>

<h3>Композиція функцій (Function Composition)</h3>

<p>
  <span class="accent">Композиція функцій</span> — це процес об'єднання двох або більше функцій для створення нової функції.
  <br>
  Результат композиції — це функція, яка виконує всі функції по порядку, передаючи результат
  однієї функції в іншу.
</p>

<p>
  Уявіть, що у нас є два прості завдання: додати до числа 10 і перетворити результат у рядок із валютою.
</p>

<code class="code">
  const addTen = (n: number): number => n + 10;
  const formatPrice = (n: number): string => `$${n}`;

  // Звичайний виклик (вкладеність)
  const result = formatPrice(addTen(5)); // "$15"
</code>

<p>
  Це і є композиція у її базовому вигляді. Але коли функцій стає 5 або 10, код перетворюється на «матрьошку» з дужок: <code>f(g(h(z(x))))</code>, що дуже важко читати.
</p>

<h3>Сompose vs Pipe</h3>
У програмуванні (особливо у функціональному підході) для цього використовують допоміжні функції, щоб перетворити вкладеність на ланцюжок.

<h4>1. Compose (справа наліво)</h4>

<p>Виконує функції у порядку, зворотному до їхнього написання (як у математиці).</p>

<code class="code">
  const compose = &lt;T>(f: (arg: T) => T, g: (arg: T) => T) => (x: T) => f(g(x));

  const complexAction = compose(formatPrice, addTen);
  console.log(complexAction(5)); // "$15"
</code>

<h4>2. Pipe (зліва направо)</h4>
<p>Більш інтуїтивний варіант для розробників: дані течуть зверху вниз або зліва направо. Подібне ви могли зустрічати в RxJS (який активно використовується в Angular).</p>

<code class="code">
  const result = pipe(
    5,
    addTen,      // 15
    formatPrice  // "$15"
  );
</code>
