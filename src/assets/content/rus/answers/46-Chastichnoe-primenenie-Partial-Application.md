<h3>Частичное применение (Partial Application)</h3>

<p>
  <span class="accent">Частичное применение</span> — это процесс создания новой функции путем
  фиксации (предустановки) одного или нескольких аргументов существующей функции.
</p>
<p>
  Простыми словами: у нас есть функция, принимающая несколько аргументов. Мы "замораживаем"
  часть из них и получаем новую функцию, которая принимает только оставшиеся аргументы.
</p>

<p class="info info--blue">
  Частичное применение основано на каррировании, но не требует всех аргументов.
</p>

<code class="code">
  function curry(f) { // curry(f) выполняет каррирование
    return function(a) {
      return function(b) {
        return f(a, b);
      };
    };
  }

  function sum(a, b) {
    return a + b;
  }

  const curriedSum = curry(sum);
  const plusTwo = curriedSum(2); // возвращает функцию, которая ждет второй аргумент

  alert( plusTwo(3) ); // 5
</code>

<h4>Практические применения частичного применения:</h4>

<ul>
  <li><strong>Упрощение повторного использования функций:</strong> Частичное применение позволяет создавать специализированные функции на основе более общих, что упрощает повторное использование кода.</li>
  <li><strong>Улучшение читаемости:</strong> Функции с фиксированными аргументами могут быть более понятными и удобными для чтения, особенно если они используются в нескольких местах.</li>
  <li><strong>Создание новых функций из существующих:</strong> Частичное применение позволяет легко создавать новые функции, не изменяя исходную функцию, что способствует более гибкому дизайну кода.</li>
</ul>

<h3>Композиция функций (Function Composition)</h3>

<p>
  <span class="accent">Композиция функций</span> — это процесс объединения двух или более функций для создания новой функции.
  <br>
  Результат композиции — это функция, которая выполняет все функции по порядку, передавая результат
  одной функции в другую.
</p>

<p>
  Представь, что у нас есть две простые задачи: прибавить к числу 10 и превратить результат в строку с валютой.
</p>

<code class="code">
  const addTen = (n: number): number => n + 10;
  const formatPrice = (n: number): string => `$${n}`;

  // Обычный вызов (вложенность)
  const result = formatPrice(addTen(5)); // "$15"
</code>

<p>
  Это и есть композиция в самом базовом виде. Но когда функций становится 5 или 10, код превращается в «матрёшку» из скобок: <code>f(g(h(z(x))))</code>, что очень тяжело читать.
</p>

<h3>Сompose vs Pipe</h3>
В программировании (особенно в функциональном подходе) для этого используют вспомогательные функции, чтобы превратить вложенность в цепочку.

<h4>1. Compose (справа налево)</h4>

<p>Выполняет функции в порядке, обратном их написанию (как в математике).</p>

<code class="code">
  const compose = &lt;T>(f: (arg: T) => T, g: (arg: T) => T) => (x: T) => f(g(x));

  const complexAction = compose(formatPrice, addTen);
  console.log(complexAction(5)); // "$15"
</code>

<h4>2. Pipe (слева направо)</h4>
<p>Более интуитивный вариант для разработчиков: данные текут сверху вниз или слева направо. Подобное ты мог встречать в RxJS (который активно используется в Angular).</p>

<code class="code">
  const result = pipe(
    5,
    addTen,      // 15
    formatPrice  // "$15"
  );
</code>
